Relationship Tree Layout

How should the relationship tree render in the workbench tab?

A

Indented Tree (like file explorer)

Classic tree with indent levels, expand/collapse markers, and resource info on each line

Relations: nginx-deployment (Deployment)
▼ Deployment nginx-deployment default Ready ▼ ReplicaSet nginx-deployment-7fb96c846b default 3/3 ├ Pod nginx-deployment-7fb96c846b-abc12 Running ├ Pod nginx-deployment-7fb96c846b-def34 Running └ Pod nginx-deployment-7fb96c846b-ghi56 Running ▶ ReplicaSet nginx-deployment-5d4b8c7f9 default 0/0
B

Bidirectional Tree (up + down from root)

Shows both "owned by" (upward) and "owns/selects" (downward) from the focused resource

Relations: nginx-deployment-7fb96c846b-abc12 (Pod)
── Owners ────────────────────────────────── Deployment nginx-deployment default ReplicaSet nginx-deployment-7fb96c846b default ── This Resource ──────────────────────────── Pod nginx-deployment-7fb96c846b-abc12 Running ── Selected By ────────────────────────────── Service nginx-svc default (selector: app=nginx) Ingress nginx-ingress default
C

Chain View (left-to-right flow)

Shows relationship chains as horizontal flows, one chain per section

Relations: nginx-svc (Service)
── Ingress Chain ──────────────────────────── Ingress nginx-ingress → Service nginx-svc ── Backend Pods ───────────────────────────── Service nginx-svc → Endpoints nginx-svc ├→ Pod nginx-abc12 Running ├→ Pod nginx-def34 Running └→ Pod nginx-ghi56 Running ── PVC Bindings (none) ──────────────────────

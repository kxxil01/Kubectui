# Specification: Per-View Cached Formatted Cells

**Track ID:** cached-cells_20260312
**Type:** Feature
**Created:** 2026-03-12
**Status:** Draft

## Summary

Cache expensive derived strings and formatted cells per view, keyed by snapshot version, to eliminate redundant formatting work on every render cycle.

## Context

Kubectui renders 23 resource views with column registries. Each render cycle recomputes formatted cell values (status strings, age calculations, resource quantities) even when the underlying data hasn't changed. This is the second phase of the M15 performance track.

## User Story

As a platform engineer, I want cached formatted cells so that table views render faster on clusters with many resources.

## Acceptance Criteria

- [ ] Correct invalidation on snapshot/query changes
- [ ] No stale data rendering
- [ ] Net positive median render improvement (5-run profiling protocol)
- [ ] Bounded memory with LRU/eviction policy
- [ ] Time-sensitive fields (Age) handled correctly with minute-bucket invalidation

## Dependencies

None. Can start immediately on current codebase (M15 already complete).

## Out of Scope

No visual or UI changes — this is a pure performance optimization behind the scenes. Row virtualization (Phase 1) and regression cleanup (Phase 3) are separate tracks.

## Technical Notes

- Cache key: `(view, query, snapshot_version, data_fingerprint)`
- Cache value: preformatted/derived cell payloads
- Age fields: use minute-bucket invalidation or render-time lightweight update path
- Keep logic centralized to avoid duplicated per-view cache behavior

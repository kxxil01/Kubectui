## Overview

Address two remaining performance bottlenecks: (1) separate semaphores for core vs secondary resource fetches to eliminate wave contention, and (2) Arc-wrap ClusterSnapshot to eliminate deep clone on every refresh cycle. Conservative approach — the Arc change is the riskier one and is attempted second.

## Phase 1: Wave Priority — Separate Fetch Semaphores

Eliminate semaphore contention between wave 1 (core) and wave 2 (secondary) by giving each its own semaphore pool.

**Current state:** Single `RESOURCE_FETCH_SEMAPHORE` with 8 permits shared across all 36 resources (`state/mod.rs:28`). Wave 2's 22 fetches can starve wave 1's 14 core fetches.

**Target:** Two semaphores — `CORE_FETCH_SEMAPHORE` (8 permits) and `SECONDARY_FETCH_SEMAPHORE` (4 permits). Core resources always get full parallelism; secondary resources use a smaller pool. Total max concurrent: 12 requests.

## Phase 2: Arc-Wrap ClusterSnapshot

Eliminate the deep clone of `GlobalState` (30+ Vec fields) at every refresh cycle by Arc-wrapping `ClusterSnapshot`.

**Current state:** `spawn_refresh_task` takes `GlobalState` by value (line 825). Call site clones: `global_state.clone()` (line 887). This deep-copies 30+ `Vec<T>` fields (500KB-2MB) synchronously in the main event loop.

**Target:** `GlobalState.snapshot` becomes `Arc<ClusterSnapshot>`. The clone at line 887 becomes an atomic refcount bump. The actual deep copy (via `Arc::make_mut`) happens asynchronously inside the spawned task when it needs mutable access — no longer blocking the event loop.

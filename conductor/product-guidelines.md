# Product Guidelines

## Voice and Tone

- Concise and direct
- Lead with the answer, not the reasoning
- Use bullet points for lists, skip unnecessary preamble

## Design Principles

1. **Performance first** — Speed and responsiveness above feature count
2. **Keyboard-first** — Every action reachable without a mouse
3. **No regressions** — Existing behavior must not break when adding features
4. **Simple over clever** — Prefer explicit, readable code over dense abstractions
